## Scaffold (Run Once to Create Project)

```bash
# 1. Create Next.js app
npx create-next-app@latest [project-name] --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"

# 2. Enter project
cd [project-name]

# 3. Install Supabase + common dependencies
npm install @supabase/supabase-js @supabase/ssr
npm install -D prisma @prisma/client

# 4. Initialize Prisma
npx prisma init --datasource-provider postgresql

# 5. Set up environment
cp .env.example .env.local   # fill in NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY
```

## CI/CD Pipeline (GitHub Actions)

```yaml
# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm run lint
      - run: npx tsc --noEmit
      - run: npx jest --coverage
      - run: npm run build

  pr-size:
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Check PR size
        run: |
          LINES=$(git diff --stat origin/main...HEAD -- . ':!package-lock.json' ':!*.generated.*' | tail -1 | awk '{print $4+$6}')
          if [ "${LINES:-0}" -gt 400 ]; then
            echo "::error::PR exceeds 400-line limit ($LINES lines changed). Split into smaller PRs."
            exit 1
          fi
```

PR size limit: 400 lines (excludes `package-lock.json` and generated files).
See `rules/git-workflow.md` for full PR standards.

## .env.example Template

```bash
# Supabase
NEXT_PUBLIC_SUPABASE_URL=your-supabase-project-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-supabase-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key   # server only, never expose to client

# Database (Prisma)
DATABASE_URL=postgresql://postgres:[password]@[host]:5432/[db]
DIRECT_URL=postgresql://postgres:[password]@[host]:5432/[db]   # for migrations

# App
NEXTAUTH_SECRET=your-nextauth-secret
NEXTAUTH_URL=http://localhost:3000
```
