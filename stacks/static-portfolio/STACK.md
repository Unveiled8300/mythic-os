## Scaffold (Run Once to Create Project)

```bash
# 1. Create Astro project (select: Empty template, TypeScript: Strict, Tailwind: yes)
npm create astro@latest [project-name]

# 2. Enter project
cd [project-name]

# 3. Add Tailwind (if not selected during setup)
npx astro add tailwind

# 4. Add image optimization
npx astro add image

# 5. Optional: add MDX for blog/content-heavy pages
npx astro add mdx
```

## CI/CD Pipeline (GitHub Actions)

```yaml
# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npx astro check
      - run: npx eslint . --ext .ts,.tsx,.astro
      - run: npm run build

  pr-size:
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Check PR size
        run: |
          LINES=$(git diff --stat origin/main...HEAD -- . ':!package-lock.json' ':!*.generated.*' | tail -1 | awk '{print $4+$6}')
          if [ "${LINES:-0}" -gt 400 ]; then
            echo "::error::PR exceeds 400-line limit ($LINES lines changed). Split into smaller PRs."
            exit 1
          fi
```

PR size limit: 400 lines (excludes `package-lock.json` and generated files).
See `rules/git-workflow.md` for full PR standards.

## Meta Tags (Required — 7 Tags in Base.astro)

Every page must include all 7 via the Base layout:

```astro
---
// src/layouts/Base.astro
const { title, description, ogImage, canonicalURL } = Astro.props;
---
<head>
  <title>{title}</title>
  <meta name="description" content={description} />
  <meta property="og:title" content={title} />
  <meta property="og:description" content={description} />
  <meta property="og:image" content={ogImage} />
  <meta property="og:url" content={canonicalURL} />
  <link rel="canonical" href={canonicalURL} />
</head>
```

Run Marketing Manager `/metadata` skill before implementing pages.
